# Testing Smart Routing Feature (Cherry Studio)

This guide provides test cases to verify the **Smart Router** feature, which automatically selects the best local model based on your prompt.

## 1. Prerequisites
- **LiteLLM Proxy**: Must be running (`./manage.sh start`).
- **Ollama**: Must be running with the required models (`qwen2.5:3b`, `qwen2.5-coder:14b`, `llama3:70b`, etc.).
- **Cherry Studio Configuration**:
  - **Provider**: OpenAI
  - **API Key**: `sk-1234` (default in `litellm_config.yaml`)
  - **Base URL**: `http://localhost:4000/v1`
  - **Model ID**: `smart-proxy`

---

## 2. Test Cases

### Test Case 1: Automatic Coding Routing
*   **Prompt**: `Write a Python script to calculate the Fibonacci sequence using recursion.`
*   **Expected Behavior**: 
    *   The Proxy logs should show: `🔄 Smart Routing: 'smart-proxy' -> 'coding'`
    *   The response should be high-quality code generated by `qwen2.5-coder`.
*   **Verification**: Check `proxy.log` for the routing message.

### Test Case 2: Automatic Reasoning Routing
*   **Prompt**: `Think step by step: If I have 3 apples and you give me 2 oranges, then I eat one apple, how many fruits do I have left and what is the logical breakdown?`
*   **Expected Behavior**:
    *   The Proxy logs should show: `🔄 Smart Routing: 'smart-proxy' -> 'reasoning'`
    *   The response should be detailed and "thoughtful," provided by `llama3:70b`.
*   **Verification**: Check `proxy.log` for the routing message.

### Test Case 3: Automatic Summarization Routing
*   **Prompt**: `Summarize the following text in three bullet points: [Paste any long article here]`
*   **Expected Behavior**:
    *   The Proxy logs should show: `🔄 Smart Routing: 'smart-proxy' -> 'summary'`
    *   The response should be a concise summary from `qwen2-vl`.
*   **Verification**: Check `proxy.log`.

### Test Case 4: Automatic Flash/Chitchat Routing
*   **Prompt**: `Hi there! How's it going?`
*   **Expected Behavior**:
    *   The Proxy logs should show: `🔄 Smart Routing: 'smart-proxy' -> 'flash'`
    *   The response should be near-instant, provided by the lightweight `glm4` model.
*   **Verification**: Check `proxy.log`.

### Test Case 5: Default Fallback (Fast)
*   **Prompt**: `What is the capital of France?`
*   **Expected Behavior**:
    *   The Proxy logs should show: `🔄 Smart Routing: 'smart-proxy' -> 'fast'`
    *   The response should be provided by `qwen2.5:3b`.

## 3. How to Monitor Routing in Real-Time

To see the "Magic" happening behind the scenes while you type in Cherry Studio, run this command in your terminal:

```bash
./manage.sh logs
```

Look for lines starting with `INFO:smart_router_hook: 🔄 Smart Routing: ...`.

## 4. Troubleshooting
- **Incorrect Routing**: If a coding task is routed to `fast`, ensure keywords like `code`, `python`, or `function` are present in your prompt.
- **Model Loading Delay**: If you see a long hang when routing to `reasoning`, it's likely Ollama loading the `70B` model into your Mac's unified memory.
